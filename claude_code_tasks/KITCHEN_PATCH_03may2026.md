**Клуб «Вкус Жизни»** — онлайн-клуб нутрициолога Натальи Томшиной.
**Стек:** Next.js + Supabase (PostgreSQL) + TypeScript
**Рабочий процесс:** Claude (этот) формулирует ТЗ → пользователь отправляет в Claude Code → SQL выполняется вручную в Supabase Studio

### Ключевые файлы кода
- `src/lib/recipeCalculator.ts` — selectRecipes, calculatePortion, getMealTarget
- `src/lib/productUtils.ts` — PRODUCT_SYNONYMS, expandProducts, PROTEIN_KEYWORDS, OFFAL_KEYWORDS, LEGUMES_KEYWORDS, VEGGIE_KEYWORDS, FAT_CAPS, getInputProteins, filterVeggies, normalizeRu

- `nutrition`: id, name, category, calories, protein, fat, carbs, name_alt

## 5. ИЗМЕНЕНИЯ В КОДЕ

### 5.1. УДАЛЕНО
- **Логика супов в рационе** — супы остаются в БД (для Умной Кухни), но в недельный план не включаются. Чекбокс «Включить супы» убран из формы. `include_soups` всегда `false` в `weekly_plans.insert`.
- **Режим `every2days` / `cook_mode`** — полностью убран. Всегда `daily`. Убраны `DayGroup`, `dayGroups`, `portions_to_cook`, `is_repeat`, `repeat_from_day`, `repeat_meal_type`. Переключатель «Режим готовки» убран из формы. Цикл по дням 1–7 напрямую.

### 5.2. МУЛЬТИ-БЕЛКОВАЯ ВЫДАЧА (weeklyPlanHelper.ts + route.ts)
- `classifyProteins(userProducts)` — разделяет на группы: freshFish, smokedFish, offal, meat, minced, eggs
- `buildProteinSchedule(proteins, mealsPerDay)` — возвращает `DayProtein[]` для 7 дней
- В route.ts для каждого дня известно: `meal1Protein` (завтрак), `meal2Protein` (обед/ужин)
- При 3-разовом обычном дне: `dinner = { ...lunch, meal_type: 'ужин' }` (приём 2 = приём 3)

### 5.3. ЛОГИКА САЛАТОВ В РАЦИОНЕ

**При `include_salads = true`:**

**2-разовое питание (7 дней):**
- vegSaladDays = [1, 3, 5] — овощной салат как добавка
- proteinSaladDays = [2, 6] — белковый салат **заменяет обед**
- noSaladDays = [4, 7] — без салата

**3-разовое питание (7 дней):**
- proteinSaladDays = [2, 5] — белковый салат на обед, **ужин отдельный** (с другим белком)
- vegSaladDays = [1, 3, 6] — овощной салат как добавка
- noSaladDays = [4, 7] — без салата

**Овощные салаты** (категория `салат` без role='protein'):
- `buildBaseKbjuMeal` — НЕ пересчитывается
- КБЖУ вычитается из дневной цели перед расчётом обеда

**Белковые салаты** (категория `салат_белковый`):
- Пересчёт через `calculatePortion` (полный масштаб)
- Заменяют lunch в proteinSaladDays
- Лимиты жиров через `FAT_CAPS` (см. ниже)

### 5.4. ОВОЩИ: разные в одном дне + охват всех за неделю
- В route.ts: `recipeVeggieMap` (recipe_id → главный овощ) для bf и main рецептов
- При выборе обеда: исключаются рецепты с тем же главным овощем что у завтрака
- `veggieUsage Map<string, number>` — счётчик использования каждого введённого овоща
- В `selectRecipes`: score +50 за неиспользованный овощ, score -10 за использованный 2+ раза
- При 3+ овощах в `user_products` — строгий фильтр: рецепт проходит только если все его veggies с `is_always_available=false` есть в expandedProds

### 5.5. УМЕНЬШЕНИЕ УГЛЕВОДНЫХ ОВОЩЕЙ (calculatePortion)
Если `totals.carbs > target.carbs + 2`:
- Найти scalable veggies (role='veggie', is_scalable_veggie=true)
- Отсортировать по убыванию carbs_per_gram
- Итеративно уменьшать каждый до минимума 30% от base_grams
- Пересчитать totals, остановиться когда target достигнут
- Если упёрлись в минимум — рецепт всё равно идёт в план (без исключения)

### 5.6. СТРОГИЙ ФИЛЬТР БЕЛКА (productUtils.ts + recipeCalculator.ts)

**Контекст:** пользователь выбирает продукты через **autocomplete** — точные значения `nutrition.name` из БД. Свободного текста нет.

**Изменения:**
- `getInputProteins(userRaw)` — теперь identity-фильтр: возвращает только те `userRaw` строки которые есть в `PROTEIN_KEYWORDS`
- `normalizeRu(s)` — хелпер: `s.toLowerCase().replace(/ё/g, 'е')`
- В `selectRecipes` для фильтра по белку: `normalizeRu(protein_tag) === normalizeRu(userProtein)` — **строгое равенство**, без `includes`/`startsWith`
- В scoring-бонусе (+10 за совпадение белка) — то же строгое равенство
- `PRODUCT_SYNONYMS` сокращён до минимально необходимых связок:
  - Разные написания одного продукта (ё/е, заглавные)
  - оливки ↔ маслины
  - масло гхи (id 506) ↔ масло топленое гхи (id 591)
  - куриная печень ↔ печень индейки (взаимозаменяемы)

**Результат:** «слабосолёная сёмга» НЕ совпадает с «сёмга», «куриная грудка» НЕ совпадает с «куриная печень», «куриный фарш» НЕ совпадает с «фарш говяжий».

## 7. КЛЮЧЕВЫЕ ПРАВИЛА АЛГОРИТМА (для будущих сессий)

### 7.1. Пользовательский ввод
**Пользователь выбирает продукты из autocomplete** — точные `nutrition.name` из БД. Свободный текст не вводится. Это значит: никакой «курицы» / «свинины» как общих понятий — только конкретные «куриная грудка без кожи», «филе куриного бедра», «сёмга слабосоленая».

### 7.2. Структура приёмов
| Режим | Структура |
|-------|-----------|
| 2-разовое | завтрак + обед/ужин (разные белки + разные овощи) |
| 3-разовое обычный день | завтрак + обед = ужин (один рецепт повторяется) |
| 3-разовое день белкового салата | завтрак + белковый салат + другой ужин (3 разных белка) |

### 7.3. Категории рецептов и где они используются
| Категория | Где используется в рационе |
|-----------|---------------------------|
| `завтрак` | приём 1 |
| `обед_ужин` | приёмы 2, 3 |
| `салат` (без protein) | овощной салат — добавка к обеду |
| `салат_белковый` | замена обеда в proteinSaladDays |
| `десерт` | bonus_desserts (3 шт, не привязаны к дням) |
| `суп` | НЕ используется в рационе (только в Умной Кухне) |

### 7.4. Жёсткие правила
- При 2+ белках — рецепты только с этими белками (никаких «чужих»)
- 3+ овощей — рецепты только с этими овощами
- Субпродукты — max 2 рецепта/неделю на каждый вид
- Свежая рыба — 2 раза в неделю (во 2-й/3-й приём)
- Копчёная/солёная рыба — без нагрева (завтрак + салат)
- target.carbs < 25 → исключить бобовые
- В лимиты салата (категория `салат_белковый`): сметана ≤ 60г, оливковое ≤ 30г, гхи ≤ 30г, майонез ≤ 40г

### 7.5. Алгоритм скоринга в selectRecipes
- Базовый: +1 за каждый совпавший тег
- +10 за совпадение `protein_tag` со строгим равенством `normalizeRu`
- +50 за неиспользованный овощ (veggieUsage = 0)
- -10 за овощ использованный 2+ раза
- +50 за неиспользованный белок (proteinUsage = 0)
- Исключение если: рецепт в excludeIds, белок-субпродукт с usage ≥ 2, при target.carbs < 25 содержит бобовое, при 3+ овощей содержит лишний veggie

## 9. ВАЖНЫЕ ЗАМЕТКИ ДЛЯ БУДУЩЕГО CLAUDE

1. **Не использовать `includes`/`startsWith` для сравнения белков** — только `normalizeRu` + `===`. Это критично!
2. **Не разворачивать PROTEIN_KEYWORDS на токены** — пользователь вводит точные имена из autocomplete.
3. **Не путать роли oil и fat:** oil масштабируется, fat фиксированный.
4. **Перед SQL про данные** — всегда сначала диагностический SELECT.
5. **Перед изменением кода** — попросить Claude Code показать план изменений до правок.
6. **Калории/КБЖУ не хранятся в recipes** — считаются на лету через nutrition.
7. **Сложные изменения в route.ts** — лучше пакетами по 3-5 пунктов, не одним мегаТЗ.
8. **При багах фильтрации** — первый шаг диагностики: смотреть `DISTINCT protein_tag` в БД (там могут быть несоответствия с PROTEIN_KEYWORDS).
