## 💳 ТАРИФЫ

- **Пробный (Триал):** 149 ₽/7 дней + рекуррент 1500₽/мес (startDate = now + 7 дней)
- **Месяц:** 1 500 ₽/мес
- **Полгода:** 6 000 ₽ разовый

### Доступ по статусам
- **trial** → марафон скрыт (только анонс-заглушка без контента), остальное доступно
- **active** → полный доступ ко всему включая чат марафона

### Уровни участниц (считаются на лету через getEffectiveMonths)
| Уровень | Стаж | Вебинары |
|---------|------|----------|
| 🌸 Новенькая | 0-3 мес | — |
| ⭐ Вошла во вкус | 3 мес | 2 на выбор |
| 🔥 Уже своя | 6 мес | +3 (итого 5) |
| 💚 Легенда | 9 мес | +2 включая курсы (итого 7) |
| 💎 Бриллиант | 12 мес | все (999) |

**Участницы «Полгода» получают +6 бонусных месяцев к стажу:**
- День 1 → «Уже своя» (5 вебинаров)
- 3 мес реальных → «Легенда» (9 мес эффективных)
- 6 мес реальных → «Бриллиант» (12 мес эффективных)

Функция: `getEffectiveMonths(createdAt, plan)` в `src/lib/webinars.ts`
⚠️ Везде передавать `subscription_started_at ?? created_at` как первый аргумент (НЕ просто created_at!)

### Бриллиант вручную (SQL для присвоения)
```sql
UPDATE members
SET
  subscription_status = 'active',
  subscription_plan = 'month',        -- ← ВАЖНО: именно 'month', не 'halfyear'!
  subscription_started_at = now() - interval '12 months',
  subscription_expires_at = now() + interval '999 years'
WHERE email IN (...);
```
⚠️ Использовать `subscription_plan = 'month'` (не halfyear!) — иначе подсказка управления подпиской не показывается.

После `ALTER TABLE` нужно либо перезапустить PostgREST, либо писать явный список полей.
**Всегда писать явный список полей** в запросах после добавления новых колонок.

### 3. Email отправка — только через mailer.ts

### 18. Supabase Realtime — новые таблицы
При создании новой таблицы которая должна работать в реальном времени — обязательно добавить в publication:
```sql
ALTER PUBLICATION supabase_realtime ADD TABLE имя_таблицы;
```
Текущие таблицы в publication: `channel_posts`, `private_messages`.

### 19. RegExp с флагом /g — не использовать на уровне модуля для .test()
При split(/(capturing group)/) совпадения на нечётных индексах (i % 2 === 1) — надёжнее.

### 20. getUser() vs getSession() — производительность
```typescript
// МЕДЛЕННО — делает сетевой вызов к GoTrue для валидации JWT:
const { data: { user } } = await supabase.auth.getUser()
// БЫСТРО — декодирует JWT из cookie локально (в 10-100x быстрее):
const { data: { session } } = await supabase.auth.getSession()
```

### 21. getEffectiveMonths — передавать subscription_started_at
```typescript
// ПРАВИЛЬНО:
getEffectiveMonths(member.subscription_started_at ?? member.created_at, member.subscription_plan)
// НЕПРАВИЛЬНО:
getEffectiveMonths(member.created_at, member.subscription_plan)
```

### 22. OTP — не отправлять expired участницам
В роуте отправки OTP проверять subscription_status. Если 'expired' → вернуть { error: 'not_found' }.
Страница /auth показывает «Участница с таким email не найдена» для этой ошибки.

### 23. webinar_selections — уникальный constraint
```sql
-- Есть constraint: webinar_selections_member_webinar_unique UNIQUE (member_id, webinar_id)
-- upsert с ignoreDuplicates: true работает корректно
```

### 24. requireAdmin в роутах — использовать email lookup
```typescript
// НЕПРАВИЛЬНО — member.id ≠ auth.users.id в этом проекте:
.eq('id', user.id)
// ПРАВИЛЬНО:
.eq('email', user.email)
```

## 💳 CLOUDPAYMENTS — ПЛАТЁЖНЫЙ ФЛОУ

```
nata-tomshina.ru (кнопка тарифа)
  → club.nata-tomshina.ru/join?plan=trial
    → ввод email → POST /api/join/save-ref (сохраняет ref_code в pending_refs)
    → CloudPayments виджет
    → оплата
    → webhook → /api/payments/cloudpayments
      → проверка HMAC подписи (verifyHmac — оба секрета)
      → создать/найти member
      → обновить subscription_status
      → начислить комиссию партнёру (если есть ref_code)
      → отправить OTP
    → successRedirectUrl → /auth?email=...&from=payment
```
