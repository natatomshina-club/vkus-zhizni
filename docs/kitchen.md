# docs/kitchen.md — Умная кухня

## Роут: /dashboard/kitchen

## Логика работы
1. Участница выбирает продукты (кнопки + текст)
2. Выбирает: 2 или 3 раза в день
3. YandexGPT составляет МЕНЮ из 2-3 РАЗНЫХ блюд
4. КБЖУ распределено между блюдами
5. Если продуктов не хватает → список покупок
6. Советы от Наташи

## Лимиты
- Триал: 3 запроса/день | Клуб: 10/день
- Поля в members: kitchen_requests_today (int4), kitchen_date (text)
- Сброс в полночь по timezone участницы

## Распределение КБЖУ
- 3-разовое: завтрак 30% / обед 40% / ужин 30%
- 2-разовое: 50% / 50%
- Белок: protein / meals_per_day на каждое блюдо

## API: POST /api/kitchen/generate
Принимает: { products, meals_per_day, userId }
Читает из members: weight, height, age, goal_weight, health_conditions, allergies, kbju_*
Возвращает: JSON меню

## UI элементы страницы
- Кнопка «📋 Примеры рационов» → /dashboard/rations
- Блок показателей: имя, КБЖУ норма, заболевания
- Счётчик запросов и избранных (X/50)
- Вопрос «Сколько раз питаешься?» → 2 / 3 раза
- 12 кнопок с эмодзи: 🥩🍗🐟🥚🧀🥦🥒🫑🥬🧈🍶🥛
- Поле ввода доп. продуктов
- Кнопка «Приготовить меню на день 🍳»
- Карточки блюд + итоговое КБЖУ + список покупок
- Блок «💬 Советы от Наташи»
- Кнопки на блюде: «⭐ Сохранить» → saved_recipes, «📓 В дневник» → diary_entries
- Последние 3 запроса
