# docs/diary.md — Дневник питания

## Роут: /dashboard/diary

## Принципы
- Заполнять необязательно — нет «провалов», нет стресса
- Пропустил день → просто пропустил, без предупреждений
- Цель: видеть динамику, не отчитываться

## 4 способа добавить приём пищи

### 1. Фото тарелки (YandexGPT Vision)
- Сфотографировал → ИИ определяет блюдо и считает КБЖУ
- «Похоже на: яичница 2 яйца + масло. ~280 ккал, Б:18г, Ж:22г, У:2г»
- Участница подтверждает или корректирует

### 2. Из Умной кухни
- Кнопка «📓 В дневник» в боте → выбрать завтрак/обед/ужин → готово
- КБЖУ уже посчитано

### 3. Из избранного / вчера
- «Повторить вчерашний завтрак?» — одна кнопка
- Или выбрать из saved_recipes

### 4. Быстрые шаблоны
- 20 частых блюд метода Натальи
- Тап → добавлено в дневник с готовым КБЖУ

## Итог дня
- Суммарное КБЖУ из всех добавленных блюд
- Прогресс-бары до нормы (из профиля)

## Умный совет от Наташи
Появляется когда добавлено ≥2 приёма пищи.
Логика порогов:
- Белок < 80% нормы → добрать (примеры продуктов)
- Белок > 130% → «отлично, завтра чуть меньше»
- Углеводы > 120% → мягкое предупреждение
- Ккал < 70% → «мало ела — замедлит результат»
- Жиры < 70г → ПЛОХО для гормонов → «добавь масло/яйца/мясо»
- Жиры 70-120г → норма
- Жиры > 120г → «чуть меньше жира завтра»
- Кнопка «🔄 Другой совет»

## Тон советов
- Всегда позитивный, НИКОГДА не осуждающий
- «Сегодня отлично! Завтра попробуй...» а не «Ты перебрала»
- Конкретные примеры: «100г куриных бёдер = +20г белка»

## Календарь
- Сетка месяца
- Дни с записями — выделены зелёным
- Клик на день → запись за тот день
- Хранение: только текст в diary_entries

## Поле «Маленькая победа»
- Необязательное, одно поле внизу дневника
- Подсказки: «Не перекусывала», «2 литра воды», «Не хотела сладкого»

## Таблица diary_entries
```sql
id, member_id, date, meal_type (breakfast/lunch/dinner/snack),
title, calories, protein, fat, carbs,
source (photo/bot/template/manual),
created_at
```

## API
- POST /api/diary/add — добавить приём пищи
- GET /api/diary/today — записи за сегодня
- GET /api/diary/[date] — записи за дату
- POST /api/diary/advice — получить совет от YandexGPT
