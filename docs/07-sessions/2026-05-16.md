# 🎯 ТЕКУЩАЯ ЗАДАЧА (НЕ СДЕЛАНА, СЛЕДУЮЩИЙ ШАГ)

## Контекст и решение

На странице `/club` (платная годовая программа) сейчас стоит цена 67 000 ₽ и кнопка прямой записи в Telegram. Хотим **скрыть всё что связано с ценой** и заменить на **анкету-диагностику**, после заполнения которой с человеком связывается помощница.

**Логика анкеты:** не отсев бедных, а диагностика по правилам SPIN-продаж + «лестница согласий». Каждый вопрос даёт человеку возможность самой себе подтвердить что (1) проблема серьёзная, (2) старые методы не работали, (3) дальше так нельзя, (4) готова инвестировать в решение, (5) возьмёт ответственность.

**Архитектурное решение:** НЕ создаём отдельную страницу `/klub` (запутаемся со ссылками на сайте и А/Б тестами). Вместо этого — **feature flag** на странице `/club`. URL остаётся тот же, переключение режима через переменную окружения. Старый код с ценой остаётся в репозитории, не удаляется. Возврат к старому варианту = смена переменной без деплоя.

### 📋 Режим «диагностика» на странице `/club`

#### Концепция

Страница `/club` остаётся на том же URL. Все ссылки на сайте, реклама, метрика продолжают работать. Внутри страницы добавляется feature flag, который переключает финальную часть воронки между двумя режимами:

- **`pricing`** — текущий вариант: блок с ценой 67 000 ₽, кнопка в Telegram, FAQ с вопросом о рассрочке
- **`diagnostic`** — новый вариант: подводящий текст + анкета-диагностика, после заполнения с человеком связывается помощница

Переключение делается одной переменной. Старый код сохраняется в репозитории.

#### Архитектура

**Feature flag — переменная окружения в `.env`:**

```bash
# Режим финальной воронки на /club
# Значения: 'pricing' (старый вариант с ценой) | 'diagnostic' (новый вариант с анкетой)
NEXT_PUBLIC_CLUB_MODE=diagnostic
```

**Утилита `src/lib/club-mode.ts`:**

```typescript
export type ClubMode = 'pricing' | 'diagnostic';

export function getClubMode(): ClubMode {
  const mode = process.env.NEXT_PUBLIC_CLUB_MODE;
  return mode === 'diagnostic' ? 'diagnostic' : 'pricing';
}
```

Дефолтное значение `pricing` на случай если переменная не задана — старый вариант работает.

**Новые файлы:**

```
src/app/public-site/club/components/

  DiagnosticForm.tsx           ← многошаговая анкета
```

#### API endpoint `/api/public/club-diagnostic-submit`

POST-эндпоинт. Принимает JSON со всеми полями анкеты.

**Валидация (Zod):**

```typescript
const submitSchema = z.object({
  age_range: z.enum(['35-45', '46-55', '56-65', '65+']),
  problems: z.array(z.string()).min(1),
  tried_methods: z.array(z.string()).min(1),
  tried_result: z.string(),
  motivation: z.string(),
  readiness: z.string(),
  name: z.string().min(2).max(100),
  contact_tg: z.string().optional(),
  contact_wa: z.string().optional(),
  contact_email: z.string().email().optional().or(z.literal('')),
  comment: z.string().max(2000).optional(),
}).refine(
  data => data.contact_tg || data.contact_wa,
  { message: 'Укажите хотя бы один способ связи', path: ['contact_tg'] }
);
```

**Логика обработки:**

1. Валидация через Zod, если ошибка — 400 с сообщением
2. Запись в БД (`klub_diagnostics`) со всеми полями
3. Отправка уведомления в Telegram Наталье (см. ниже)
4. Возврат `{ success: true }` клиенту

**Telegram-уведомление:**

Отправляется через Telegram Bot API. Нужен бот-токен (создать у `@BotFather`, положить в `.env` как `TELEGRAM_BOT_TOKEN`). Chat ID Натальи (https://t.me/NataTomshina) — получить через `@userinfobot` или через первое сообщение боту, положить в `.env` как `TELEGRAM_NATALIA_CHAT_ID`.

```typescript
async function sendTelegramNotification(data: SubmitData) {
  const message = formatDiagnosticMessage(data);
  
  await fetch(`https://api.telegram.org/bot${process.env.TELEGRAM_BOT_TOKEN}/sendMessage`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      chat_id: process.env.TELEGRAM_NATALIA_CHAT_ID,
      text: message,
      parse_mode: 'HTML',
    }),
  });
}
```

**Email НЕ отправляем** — Resend/SendGrid в проекте не работают. Только БД + Telegram.

**Обработка ошибок:**
- Если БД упала — 500, на клиенте «Что-то пошло не так, попробуйте ещё раз или напишите нам в Telegram»
- Если Telegram-уведомление не отправилось — анкета **всё равно сохраняется в БД** (не блокируем сохранение). Логируем ошибку отдельно
- Rate-limiting: одна заявка с одного IP за 60 секунд
