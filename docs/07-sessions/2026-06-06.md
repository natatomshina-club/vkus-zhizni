### ✅ Новая churn-серия (expiry-followup)

Архитектура:
```
SUBSCRIPTION_ENDS_AT < NOW()
  └─ 03:00 UTC: check-subscriptions → status='expired'
       └─ 11:00 UTC: expiry-followup → проверка day = today - ends_at
              ├─ day 1, 2, 3 → шаблон 1 «Доступ в Клуб приостановлен»
              ├─ day 15       → шаблон 2 «Скоро старт ежемесячного марафона»
              └─ day 30       → шаблон 3 «Вы ещё можете вернуться без потери статуса»
```

Новое поле `members.expiry_followup_step` (int, default 0) — хранит последний отправленный день. Сбрасывается в 0 при возобновлении подписки в трёх местах: `admin/members/[id]/route.ts` (три блока), `webhook/cloudpayments/route.ts` (PAY + RECURRENT).

Тестовый запуск: 14 expired-участниц проверено, одна (`larisa.kynicina@mail.ru`, истекла 3 июня) попала на день 3 → реальное письмо отправлено.
