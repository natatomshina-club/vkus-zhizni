## ✅ ЧТО СДЕЛАНО В ЭТОЙ СЕССИИ

### 1. HMAC защита CloudPayments вебхука
- Добавлена проверка подписи Content-HMAC в `/api/payments/cloudpayments/route.ts`
- `verifyHmac()` проверяет оба секрета: CP_API_SECRET (боевой) и CP_API_SECRET_TEST (тестовый)
- Переменные добавлены в `.env.local` и `/home/deploy/app/.env.production.local`
- Без правильной подписи → `{code: 0}` без обработки платежа

### 2. Файрвол ufw
- Включён: `ufw enable`
- Открыты только порты: 22 (SSH), 80 (HTTP), 443 (HTTPS)
- Все остальные порты закрыты

### 3. PostgreSQL и Supabase закрыты от интернета
- Проблема: Docker обходит ufw через iptables DOCKER chain
- Решение: в docker-compose.yml Supabase привязали порты к 127.0.0.1
- Закрытые порты: 5432 (PostgreSQL), 8000 (Kong), 6543 (PgBouncer), 8443 (HTTPS)
- Бэкап оригинала: `/opt/supabase/docker/docker-compose.yml.bak`

### 4. Security headers в Nginx
Добавлены в конфиг club.nata-tomshina.ru:
```nginx
add_header X-Frame-Options "SAMEORIGIN" always;
add_header X-Content-Type-Options "nosniff" always;
add_header Referrer-Policy "strict-origin-when-cross-origin" always;
add_header Permissions-Policy "camera=(), microphone=(), geolocation=()" always;
```

### 5. Валидация файлов в чате (channel-media)
- В `upload-url/route.ts`: поле `size` теперь обязательно (было опциональным)
- Bucket policy в Supabase Storage:
  ```sql
  UPDATE storage.buckets 
  SET file_size_limit = 10485760,
      allowed_mime_types = ARRAY['image/jpeg','image/png','image/webp','image/heic','image/heif','image/gif']
  WHERE id = 'channel-media';
  ```
- Двойная защита: сервер отклоняет до загрузки + Storage отклоняет при загрузке

### 6. Rate limiting на OTP
- Повтор запроса OTP не чаще 1 раза в 2 минуты
- Реализовано в роуте отправки OTP партнёрки

### 7. Cron удаления старых фото из чата
- pg_cron job: `cleanup-expired-posts`
- Расписание: ежедневно в 03:00 UTC
- Удаляет посты с истёкшим `media_expires_at` и файлы из Storage

### 11. Supabase Studio

### 12. Cloudflare Proxy — важный урок
- Включили Proxy для club.nata-tomshina.ru → CloudPayments вебхуки перестали работать
- Причина: Cloudflare блокирует запросы на "внутренние IP" (SSRF защита)
- Решение: club.nata-tomshina.ru вернули в DNS only
- ⚠️ ПРАВИЛО: club.nata-tomshina.ru ВСЕГДА должен быть DNS only!

### 13. Исправлен баг HMAC мismatch
- Причина: в .env был секрет боевого магазина, тестировали через тестовый магазин
- Решение: verifyHmac() проверяет оба секрета параллельно

## ⚠️ ВАЖНЫЕ УРОКИ СЕССИИ

1. **Cloudflare Proxy + CloudPayments не совместимы** для домена с вебхуком
2. **Docker обходит ufw** — нужно привязывать порты к 127.0.0.1 в docker-compose
3. **Разные секреты** у тестового и боевого магазинов CP — verifyHmac должен знать оба
4. **VNC консоль Beget** — спасательный круг при потере SSH доступа
5. **Всегда проверять** работу платежей после любых изменений в инфраструктуре
