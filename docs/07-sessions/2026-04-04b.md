### 1. Зелёные кружки в календаре дневника — НЕ РАБОТАЕТ ❌
**Суть:** Календарь не подсвечивает дни где есть записи в дневнике.

**Что проверили:**
- Данные в БД есть (проверено через SQL)
- `diary_entries` таблица в схеме `public` ✅
- GRANT ALL выполнен на все diary таблицы ✅
- PostgREST перезапущен ✅
- API `/api/diary/calendar` возвращает `{"markedDays":[],"feelingDays":[]}` — пустые массивы
- `createServiceClient` использует `http://supabase-kong:8000` внутри Docker
- Добавили `SUPABASE_INTERNAL_URL=http://supabase-rest:3000` в .env.production.local

**Гипотеза:** PostgREST (supabase-rest:3000) доступен изнутри Docker сети, но запросы через Kong к diary таблицам возвращают пустой результат. Возможно проблема в RLS или в том как Kong проксирует запросы от service_role.

**Файлы изменённые по этому багу:**
- `src/app/api/diary/calendar/route.ts` — переписан с логами
- `src/lib/supabase/server.ts` — добавлен SUPABASE_INTERNAL_URL fallback
- `src/components/DiaryClient.tsx` — изменён стиль подсветки дней

**Следующий шаг:** Проверить напрямую запрос от Next.js контейнера к supabase-rest:3000 минуя Kong. Или добавить console.error в calendar route и смотреть что реально возвращает supabaseAdmin запрос.
