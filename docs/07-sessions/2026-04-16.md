## ✅ Марафоны — ручное управление днями

### Проблема
Дни марафона не переключались автоматически — `starts_at` был установлен вчера, но участницы видели День 1.

### Решение
- Добавлена колонка `marathon_days.is_active BOOLEAN DEFAULT false`
- Текущий день теперь = максимальный `day_number` где `is_active = true`
- В админке `/admin/marathons/[id]/days` добавлена кнопка **«Открыть день»** для каждого дня
- При нажатии: день активируется + автосообщение в чат марафона от имени Наташи:
  > 🔥 День N марафона уже открыт! Заходите на страницу марафона...

### SQL выполнен вручную
```sql
ALTER TABLE marathon_days ADD COLUMN IF NOT EXISTS is_active BOOLEAN DEFAULT false;
UPDATE marathon_days SET is_active = true 
WHERE marathon_id = '636099cf-...' AND day_number = 1;
```

## ✅ Марафоны — навигация по прошлым дням

### Проблема
Участницы не могли посмотреть прошедшие дни — показывался только текущий день. Отстающие участницы не могли наверстать.

### Решение
- Шкала дней (кружки) стала кликабельной для всех активных дней
- Добавлен `selectedDay` state (по умолчанию = текущий день)
- При клике на прошлый день — показывается задание/рацион/комментарий того дня
- Кнопка «Я выполнила» доступна для **любого активного дня** (и прошлых и текущего)
- Если день выполнен — показывается «✅ День N засчитан!»

## ✅ Марафоны — кнопка «Я выполнила» и прогресс

### Проблема
- Кнопка «Я выполнила» давала 500 ошибку
- Счётчик прогресса не обновлялся

### Причины и фиксы
1. **member_id через user.id** → исправлено на lookup по email
2. **Нет UNIQUE constraint** → добавлен:
   ```sql
   ALTER TABLE marathon_task_completions 
   ADD CONSTRAINT marathon_task_completions_unique 
   UNIQUE (marathon_id, member_id, day_number);
   ```
3. **task_index NOT NULL** → убрано ограничение:
   ```sql
   ALTER TABLE marathon_task_completions 
   ALTER COLUMN task_index DROP NOT NULL;
   ```
4. **INSERT → upsert** с `onConflict: 'marathon_id,member_id,day_number'`
5. **Формула прогресса** исправлена:
   ```typescript
   // БЫЛО: completions.size / currentDay * 100
   // СТАЛО: completions.size / marathon.duration_days * 100
   ```
6. **Optimistic update** — прогресс обновляется мгновенно после нажатия

## ✅ HTML-презентации в вебинарах

### Что сделано
- Новая колонка `webinars.html_url TEXT` (SQL выполнен вручную)
- Роут загрузки `POST /api/admin/webinars/[id]/upload-html` — загружает `.html` в bucket `webinar-materials/{id}/presentation.html`
- Роут удаления `DELETE /api/admin/webinars/[id]/upload-html`
- В админке вебинара появился блок **«🖥 HTML-презентация»** с кнопкой «Прикрепить HTML»
- Роут-прокси `GET /api/webinars/presentation?url=...` — отдаёт HTML с правильным `Content-Type: text/html`
- Viewer-страница `/dashboard/webinars/presentation?url=...` с кнопкой «← Назад» для мобильного PWA
- Кнопка «Открыть презентацию» у участниц: десктоп → новая вкладка, мобильный/PWA → внутренний viewer

### SQL выполнен вручную
```sql
ALTER TABLE webinars ADD COLUMN IF NOT EXISTS html_url TEXT;
ALTER TABLE webinar_lessons ADD COLUMN IF NOT EXISTS html_url TEXT;
```

### Важный урок
Supabase Storage отдаёт `.html` файлы как `text/plain` — браузер показывает код.
Решение: роут-прокси который добавляет `Content-Type: text/html; charset=utf-8`.

### Определение мобильного для открытия viewer
```typescript
const isMobile = window.innerWidth < 768 || 
  window.matchMedia('(display-mode: standalone)').matches
```
