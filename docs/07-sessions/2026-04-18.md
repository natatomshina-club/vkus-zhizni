## ✅ Автосерии писем — инфраструктура

### Новые таблицы (SQL выполнен вручную)
```sql
CREATE TABLE email_sequences (
  id uuid PRIMARY KEY,
  series text NOT NULL,      -- 'welcome_members' | 'welcome_leads' | 'evergreen'
  step integer NOT NULL,
  subject text NOT NULL,
  html text NOT NULL DEFAULT '',
  delay_days integer NOT NULL DEFAULT 1,
  is_active boolean DEFAULT true,
  UNIQUE (series, step)
);

CREATE TABLE subscriber_sequence_progress (
  id uuid PRIMARY KEY,
  subscriber_id uuid REFERENCES subscribers(id),
  series text NOT NULL,
  current_step integer DEFAULT 0,
  next_send_at timestamptz,
  completed boolean DEFAULT false,
  UNIQUE (subscriber_id, series)
);

CREATE TABLE email_templates (
  id uuid PRIMARY KEY,
  name text NOT NULL,
  subject text NOT NULL DEFAULT '',
  html text NOT NULL DEFAULT '',
  created_at timestamptz DEFAULT now()
);
```

### Серии писем
- `welcome_members` — для всех новых участниц (триал + месяц), стартует при первой оплате
- `welcome_leads` — для лидов с сайта, стартует при подписке
- `evergreen` — вечнозелёная, стартует автоматически после `welcome_leads`

### Защита от дублей
Перед стартом серии проверяется: есть ли уже запись в `subscriber_sequence_progress` для этого подписчика + серии. Если есть — не запускать повторно. Это защищает от повторного старта когда триал переходит в месяц.

## ✅ Вебинары — раздельный счёт платных и бесплатных

### Проблема
Вебинары открытые Наташей вручную (платные) засчитывались в бесплатную квоту по уровню → участница видела «Осталось: 0» хотя квоту не использовала.

### Решение
Новая колонка `webinar_selections.is_paid`:
```sql
ALTER TABLE webinar_selections ADD COLUMN IF NOT EXISTS is_paid boolean DEFAULT false;
UPDATE webinar_selections SET is_paid = true WHERE status = 'granted';
```

- Открыто Наташей через запрос → `is_paid = true` (не считается в квоту)
- Выбрано участницей бесплатно → `is_paid = false` (считается в квоту)

### Важный урок
Страница вебинаров — Server Component, читает БД напрямую в `page.tsx`. API роут `/api/webinars` этой страницей НЕ используется. Изменения нужно вносить в `page.tsx`, а не только в API.
