## 🚨 КРИТИЧЕСКИ ВАЖНОЕ

### Два высокоприоритетных бага в email-системе (нашли в этой сессии)

**R55 — серии писем уходят со сломанной ссылкой «Отписаться»**

`cron/email-sequences/route.ts` вызывает `sendEmail()` **без замены `{{unsubscribe_token}}`** → все автоматические серии (`welcome_leads`, `evergreen`, `welcome_members`) приходят с буквальной строкой `{{unsubscribe_token}}` в ссылке. Получатели не могут отписаться.

**Это нарушает требования законодательства об отписке.** Чинить срочно.

Фикс: добавить в код по аналогии с `send-announcement/route.ts:104`:
```typescript
const personalHtml = html.replace('{{unsubscribe_token}}', emailToToken(email))
sendEmail({ ...params, html: personalHtml, raw: true })
```

**R56 — VIP-напоминания приходят битыми**

`cron/subscription-reminders/route.ts` шлёт полный `<!DOCTYPE html>` без `raw: true` → `sendEmail` оборачивает его ещё раз в `baseEmailTemplate` → получается вложенный HTML-документ внутри `<body>`.

С момента запуска cron 19 мая VIP-участницы получают битые письма.

Фикс: добавить `raw: true` в вызов `sendEmail()`.

## 📌 Методика (не меняется)

1. **Код = первый источник истины.** Документация — вторичный.
2. **ТЗ → разведка в чате → ОК → запись → git status → коммит.** Не пропускать шаги.
3. **git status после каждого шага** — только ожидаемые файлы.
4. **Никаких изменений на проде** без явного «делай» от Наташи.
5. **Один модуль за итерацию** — не смешивать задачи.
6. **Секреты не пишем** — только имена переменных.
7. **Попутные находки** — в копилку `_findings.md` (правило в CLAUDE-RULES.md).
8. **Коммитим логическими блоками** — не «всё разом», но и не «после каждой строки».
