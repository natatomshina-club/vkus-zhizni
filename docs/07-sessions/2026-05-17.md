## ✅ Фикс 1 — Race condition в PDF-превью «Я и моё тело»

### Проблема
В `/dashboard/body` при выборе любого фильтра PDF-уроки оставались в списке, но превью PDF не рендерилось — оставалась только кнопка «Скачать». Без фильтра всё работало.

### Диагноз
`PdfViewerPage` использовал `useSearchParams()` внутри Suspense. При client-side навигации (через `<Link>`) Next.js сначала рендерит Suspense fallback (пустой), потом монтирует компонент. В этот момент `useSearchParams()` мог вернуть `null` → `url` пустой → `viewerUrl` пустой → `iframe` не рендерится. При hard navigation (перезагрузке) гонки не было.

Фильтр сам по себе ни при чём — он просто добавлял лишний шаг навигации, повышавший вероятность попасть в race condition.

### Решение
- `src/app/(club)/dashboard/body/pdf/page.tsx` — серверный компонент, читает `searchParams` из пропсов (серверная сторона, без гонок)
- `PdfViewer` остался клиентским, но получает `url` как проп — готовый к рендеру
- Превентивно тот же фикс применён к `src/app/(club)/dashboard/webinars/presentation/page.tsx`
- Добавлен fallback «Файл недоступен» / «Презентация недоступна» если `url` пустой

### Статус
✅ Задеплоено

## ✅ Фикс 3 — `subscription_plan` для полугодового тарифа

### Проблема
Участницы, купившие полугодовой тариф (6000₽), в админке отображались со статусом «🌸 Новенькая» вместо «🔥 Уже своя».

### Диагноз
Webhook CloudPayments (`src/app/api/payments/cloudpayments/route.ts`) записывал план в колонку `tariff`, но НЕ в `subscription_plan`. А функция `getEffectiveMonths()` в `src/lib/webinars.ts` читает именно `subscription_plan`:

```typescript
const bonus = (plan === 'halfyear' || plan === 'Полгода') ? 6 : 0
```

→ `subscription_plan = NULL` → бонус +6 мес не добавлялся → 0 эффективных месяцев → «Новенькая».

### Решение
- В webhook `PAY` и `RECURRENT` блоки добавлена строка `subscription_plan: plan` рядом с существующим `tariff: plan`
- `tariff` оставлен для обратной совместимости
- Защита: `resolvePlan()` имеет ранний `return` при `null`, поэтому пустое значение никогда не запишется
- Для ручных бриллиантов рекуррент 1500₽ пишет `subscription_plan: 'month'` поверх существующего `'month'` — статус не сломается, считается от `subscription_started_at` (12 мес назад)

### SQL-миграция (выполнена вручную в Supabase Studio)
```sql
UPDATE members
SET subscription_plan = 'halfyear'
WHERE last_payment_amount = 6000
  AND subscription_plan IS NULL
  AND subscription_expires_at > now();
-- UPDATE 4
```

## 🔍 Расследование — ежедневные письма «Доступ в клуб приостановлен»

### Изначальная задача

### Изменение задачи в процессе
Запросили просто остановить ежедневную отправку (новую серию делать пока не нужно).

### Что обнаружено в процессе расследования

**OTP клуба идёт через Resend, а не через Beget** — это критическая зависимость, о которой никто не знал. Карта:

| Поток | Механизм | Куда идёт |
|---|---|---|
| Клубный OTP (`/auth`) | `supabase.auth.signInWithOtp()` → Supabase GoTrue → `smtp.resend.com` | ⚠️ Resend |
| OTP после оплаты | то же самое | ⚠️ Resend |
| OTP на `/join` | то же самое | ⚠️ Resend |
| Рацион OTP | `sendEmail()` → nodemailer → Beget | ✅ Beget |
| Партнёрский OTP | `sendEmail()` → nodemailer → Beget | ✅ Beget |

**Письма «Доступ в клуб приостановлен» отправляются через Resend Email API**, но **не нашим кодом**:
- В `email_campaign_logs` за последние 3 дня — 0 записей
- В коде `grep` по тексту письма — ничего
- В Resend → Audiences/Broadcasts/Automations — везде пусто
- Получатель `916995@mail.ru` отсутствует в `members` и `subscribers`
- API keys в Resend: `vkus-zhizni-new` (Last used 1 час назад — это OTP), `vkus-zhizni` (Last used 2 мес назад), `Onboarding` (No activity)
- Все письма «приостановлен» в Resend Sending logs — реальные

**Гипотезы:**
- Скрытый сервис или старая инстанция приложения (не подтверждено)
- Источник в коде, который не находится через `grep` — возможно через прямой `fetch('https://api.resend.com/emails', ...)` без слова `resend` в окрестностях

### Решение
Расследование отложено. Получатели — тестовые свои люди, жалоб нет, на бесплатный план Resend (100/день, 3000/мес) лимит не давит.

### Что осталось в наследии
- DNS поддомена `send.nata-tomshina.ru` в Cloudflare указывает на Amazon SES (Resend под капотом):
  - `MX send → feedback-smtp.ap-northeast-1.amazonses.com`
  - `TXT send → v=spf1 include:amazonses.com`
  - `TXT resend._domainkey → p=MIGfMA0...`
- API keys в Resend не отозваны (нужны для OTP)
- `RESEND_API_KEY` в `.env.production.local` (используется для `src/app/api/join/track-email/route.ts` — добавляет email в Resend Audience, но `RESEND_AUDIENCE_ID` пуст, поэтому блок `if (audienceId)` не выполняется)
