```sql
id, title, category, cooking_method, time_minutes,

```

## 🧮 Алгоритм расчёта порции

Файл `src/lib/recipeCalculator.ts` уже создан — положи его в проект.

### Суть алгоритма (объяснение для понимания):

**Роли ингредиентов:**
- `protein` — главный белковый продукт (курица, рыба, говядина, яйца, творог). Граммы считаются **динамически** под белок участницы
- `veggie` — овощи. Граммы **фиксированные** (они мало влияют на белок/жир)
- `fat` — дополнительный жир: сливки, авокадо, сыр. Граммы **фиксированные**
- `oil` — масло (гхи или оливковое). Граммы считаются **динамически** — добирает жиры до нормы
- `spice` — специи. Граммы фиксированные, КБЖУ не считается

**Порядок расчёта для одной участницы:**
1. Берём суточные макросы из `members` (kbju_protein, kbju_fat, kbju_carbs)
2. Делим на количество приёмов пищи (2 или 3) → получаем цель на один приём
3. Считаем белок от фиксированных ингредиентов (овощи, сливки и т.д. тоже дают белок!)
4. Остаток белка = цель - белок_от_фиксированных → вот сколько нужно от белкового продукта
5. Считаем граммы белкового продукта: `граммы = остаток_белка / белок_на_100г * 100`
6. Считаем жир от всех ингредиентов кроме масла
7. Добавляем масло чтобы добрать жир до цели: `граммы_масла = (цель_жир - текущий_жир) / (жир_масла/100)`
8. Углеводы считаются по остаточному принципу — просто суммируем что получилось
9. Допустимое отклонение ±2г по белку и жирам

**Пример:**
```
Участница: Б:75г / Ж:95г / У:80г, питание 3 раза
Цель на приём: Б:25г / Ж:32г / У:до 27г

Рецепт "Завтрак с яйцами и сёмгой":
- Яйца (protein): нужно 25г белка
  - Авокадо 40г даёт 0.8г белка
  - Айсберг 60г даёт 0.5г белка  
  - Огурец 50г даёт 0.4г белка
  - Сёмга 40г даёт 8г белка
  - Итого от фиксированных: 9.7г белка
  - Остаток для яиц: 25 - 9.7 = 15.3г белка
  - Граммы яиц: 15.3 / 12.9 * 100 = 119г (≈2 яйца)
- Жир от яиц+авокадо+сёмги+овощей: 24г
- Масло оливковое: (32 - 24) / 0.998 = 8г
→ Итог: Б 25.1 / Ж 31.9 / У 7.9 ✅
```

## 🔌 API роут

### `POST /api/kitchen/recipes`

**Принимает:**
```typescript
{
  category: 'завтрак' | 'обед_ужин' | 'салат' | 'десерт' | 'суп',
  meals_per_day: 2 | 3,
  user_products: string[]  // ['курица', 'брокколи', 'яйца']
}
```

**Логика роута:**
```typescript
// 1. Получаем профиль участницы
const member = await supabase
  .from('members')
  .select('kbju_protein, kbju_fat, kbju_carbs, kitchen_requests_today, kitchen_requests_date, subscription_status')
  .eq('id', userId)
  .single()

// 2. Проверяем лимит запросов
const limit = member.subscription_status === 'trial' ? 3 : 10
if (member.kitchen_requests_today >= limit) {
  return { error: 'limit_reached', limit }
}

// 3. Считаем цель на приём
const target = getMealTarget(
  { protein: member.kbju_protein, fat: member.kbju_fat, carbs: member.kbju_carbs },
  meals_per_day
)

// 4. Берём все рецепты нужной категории с тегами
const { data: allRecipes } = await supabase
  .from('recipes')
  .select('id, title, category, tags, tip_tags')
  .eq('category', category)
  .eq('is_active', true)

// 5. Выбираем 3 подходящих рецепта по тегам
const recipeIds = selectRecipes(allRecipes, user_products, category, [], 3)

// 6. Загружаем полные данные рецептов с ингредиентами
const { data: recipes } = await supabase
  .from('recipes')
  .select(`
    id, title, category, steps, tip_tags,
    recipe_ingredients (
      nutrition_id, ingredient_name, role, base_grams, is_always_available,
      nutrition ( id, name, calories, protein, fat, carbs )
    )
  `)
  .in('id', recipeIds)

// 7. Пересчитываем порции под участницу
const results = recipes.map(recipe =>
  calculatePortion(recipe, target, user_products)
)

// 8. Находим совет по тегам рецептов
const allTipTags = recipes.flatMap(r => r.tip_tags ?? [])
const { data: tip } = await supabase
  .from('cooking_tips')
  .select('tip_short')
  .in('tag', allTipTags)
  .eq('is_active', true)
  .limit(1)
  .single()

// 9. Инкрементируем счётчик
await supabase.from('members').update({
  kitchen_requests_today: member.kitchen_requests_today + 1,
  kitchen_requests_date: new Date().toISOString().split('T')[0]
}).eq('id', userId)

// 10. Возвращаем
return {
  recipes: results,
  tip: tip?.tip_short ?? null,
  requests_left: limit - member.kitchen_requests_today - 1
}
```

**Возвращает:**
```typescript
{
  recipes: [
    {
      recipe_id: number,
      title: string,
      category: string,
      steps: string[],
      ingredients: [{ name, grams, calories, protein, fat, carbs }],
      total: { calories, protein, fat, carbs },
      extra_products: string[],  // что нужно докупить
      macros_ok: { protein, fat, carbs }
    }
  ],
  tip: string | null,           // один совет по приготовлению
  requests_left: number
}
```

## ⚙️ Технические детали

### Сброс счётчика запросов
В API роуте проверять дату последнего запроса:
```typescript
const today = new Date().toISOString().split('T')[0]
if (member.kitchen_requests_date !== today) {
  // новый день — сбрасываем счётчик
  requests_today = 0
}
```

### Лимиты
- Триал: 3 запроса/день
- Полный клуб: 10 запросов/день

### Сохранение рецепта (кнопка «Сохранить»)
```typescript
// POST /api/kitchen/save
// Сохраняет в таблицу saved_recipes (уже существует)
{
  recipe_id, title, ingredients_json, total_kbju_json
}
```

### Добавление в дневник (кнопка «В дневник»)
```typescript
// POST /api/diary/add (уже существует)
{
  meal_type: 'breakfast' | 'lunch' | 'dinner',
  title: recipe.title,
  calories: recipe.total.calories,
  protein: recipe.total.protein,
  fat: recipe.total.fat,
  carbs: recipe.total.carbs,
  source: 'bot'
```
